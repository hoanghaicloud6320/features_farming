HIGH-PRIORITY FARMER TODO: PRESERVE PER-SIBLING PROVENANCE
- Current route farming can generalize semantic siblings:
    GET  /v1/admin/session
    GET  /v1/admin/licenses
    GET  /v1/admin/audit-logs
      -> GET /v1/admin/:var
- Concrete endpoint names remain recoverable from endpoint examples, so Gemini
  can unroll them when explicitly instructed.
- However, cross-session features currently aggregate request fields, response
  schemas, query keys, and status counts at the :var family level.
- This means features are sufficient for concrete endpoint discovery but not
  for lossless per-sibling contract attribution.
- Preserve a member index under every generalized route family:
    concrete method + path
    per-member request count and session/iteration support
    per-member observed status counts
    per-member query keys
    per-member request fields and response schemas
    per-member examples and relations
- Keep the generalized family as the primary abstraction, but make unrolling
  lossless and explicitly warn when an attribute exists only at family level.
- Add a regression benchmark where siblings deliberately have different
  statuses and incompatible request/response schemas.
